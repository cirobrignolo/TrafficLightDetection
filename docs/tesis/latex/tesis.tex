\chapter*{Resumen}
\addcontentsline{toc}{chapter}{Resumen}

Los sistemas de conducción autónoma representan infraestructuras de software complejas que integran múltiples componentes de machine learning. El testing exhaustivo de estos sistemas es fundamental para garantizar su confiabilidad y seguridad, pero la complejidad de los sistemas completos dificulta el diseño de tests específicos y controlados.

Este trabajo presenta una metodología de testing basada en la reimplementación de subsistemas críticos de vehículos autónomos. Como caso de estudio, se realizó el análisis mediante reverse engineering del módulo de detección de semáforos del sistema Apollo de Baidu, seguido de su reimplementación en PyTorch. Esta reimplementación sirvió como plataforma controlada para comprender profundamente el sistema y diseñar tests exploratorios que identificaran debilidades y comportamientos inesperados.

El proceso incluyó: (1) dos aproximaciones de implementación, la primera fallida intentando recortar Apollo y la segunda exitosa mediante reimplementación desde cero, (2) análisis iterativo del código fuente para extraer especificaciones funcionales, (3) validación de equivalencia mediante revisión a nivel código, y (4) descubrimiento de errores de comprensión y diseño de tests basados en el conocimiento interno del sistema.

Los resultados demuestran la viabilidad de este enfoque iterativo para comprender y testear sistemas ML complejos, revelando tanto errores de implementación como decisiones de diseño del sistema original que serían difíciles de descubrir mediante testing de caja negra.

\textbf{Palabras clave:} testing de software, sistemas de conducción autónoma, reverse engineering, oracle-based testing, detección de semáforos, machine learning, Apollo, PyTorch
